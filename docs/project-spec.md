# プロジェクト概要

Slug-Free Permalinks は、選択した公開投稿タイプと公開タクソノミーのパーマリンクを、スラッグではなく ID を使う形式へ切り替える WordPress プラグインです。`/post/123/` と `/post-123/` を選べます。

## 公開向けプロダクト情報

### 一言でいうと

スラッグ管理を減らしたい WordPress 管理者向けに、選んだ投稿・分類を ID ベース URL に切り替えるプラグイン。

### キャッチコピー

- 課題解決型: 投稿のたびにスラッグを考える作業を、URL 設計から減らす。
- 得られる結果型: タイトルや言語に左右されにくい、ID ベースの URL へ。
- 利用シーン型: 新しい WordPress サイトのパーマリンク方針を決めるときに。

### 誰のどんな困りごとを解決するか

最優先の利用者は、新規サイトまたはパーマリンク方針を検討中の WordPress 管理者です。投稿やカテゴリー、タグを公開するたびにスラッグを決めたり、タイトル変更後に URL とタイトルのずれを気にしたりする運用を減らします。マルチバイト文字が URL エンコードで長く読みにくくなる場面にも対応します。

### 提供する価値

- 投稿・分類の URL をタイトルや言語固有のスラッグに依存しない ID ベース形式へ揃えられます。
- どの公開投稿タイプ・公開タクソノミーに適用するかを選べるため、対象外の URL は従来どおりに保てます。
- タイトルを変更しても URL の ID は変わらないため、URL と表示タイトルを別々に管理できます。

### 代表機能

- 公開投稿タイプを個別選択 → 選んだ投稿タイプだけを ID ベース URL に切り替えられます。
- 公開タクソノミーを個別選択 → 選んだカテゴリーやタグなどだけを同じ形式にできます。
- スラッシュ形式・ハイフン形式の選択 → `/post/123/` または `/post-123/` を選べます。
- 旧スラッグ URL の任意リダイレクト → WordPress が解決できる旧 URL を、現在の ID ベース URL へ転送できます。
- 既存の URL 接頭辞を保持 → 他のプラグインが付与した `/en/` などの接頭辞を ID ベース URL に引き継げます。

### 選ばれる理由

- すべての投稿タイプを一律に変えるのではなく、設定画面で選んだ公開投稿タイプと公開タクソノミーだけに適用します。
- 旧 URL の転送では、404 ごとのスラッグ推測を行わず、WordPress がすでに解決できるリクエストだけを対象にします。
- URL 形式を変更することに集中したプラグインで、コンテンツ機能やフロントエンド UI は追加しません。

### 利用シーン

- 新しい WordPress サイトで、スラッグを運用ルールに含めず URL 方針を決めたいとき。
- カスタム投稿タイプやタクソノミーを使い、対象ごとに ID ベース URL を設定したいとき。
- 言語ディレクトリなどの URL 接頭辞を使いながら、対象コンテンツの URL を ID 形式にしたいとき。

### 制約・対象外

- 固定ページは対象外です。
- 投稿タイプのスラッグとタクソノミーのスラッグが同じ場合、ID ベースのリライトルールが競合する可能性があります。
- 既存のスラッグ URL を多く公開しているサイトでは、有効化前に被リンク、検索流入、SNS 共有、編集フローへの影響を確認してください。
- 旧 URL のリダイレクトは任意設定であり、WordPress が解決できない 404 URL を推測して転送する機能はありません。
